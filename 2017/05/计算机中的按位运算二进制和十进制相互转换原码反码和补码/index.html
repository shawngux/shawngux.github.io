计算机中的按位运算、二进制和十进制相互转换、原码，反码和补码
Friday, May 19, 2017

在计算机底层中，所有的运算都是采用二进制进行。所谓二进制，就是指逢二进一。二进制数由0、 1组成。

一个二进制数0或1在底层叫作1”位”。其中八个0或1组成的8位代表一个字节。

我们平时看到的字母、数字、汉字等在都是由字节组成，到计算机最底层就是由”位”来表示。

一般：

  • 一个整数占4个字节，一个字节由8组成，因此一个整数由32表示。
  • 一个英文字母占1个字节，由8表示。
  • 一个汉字占2个字节，由16位表示。

按位运算

按位运算简单来说就是对位进行操作。

按位与（&）

按位与（&）运算：两个数相同位上的（二进制数）都为1，那么运算结果的相同位上的数也为1，其他都为0。

x = 00000000 00000000 00000000 00001010

y = 00000000 00000000 00000000 00110010

x & y = 00000000 00000000 00000000 00000010

按位或（|）

按位或(｜)运算：两个数相同位上的（二进制数）只要有一个是1，那么运算结果的相同位上的数就为1，否则为0。

x = 00000000 00000000 00000000 00001010

y = 00000000 00000000 00000000 00110010

x ｜ y = 00000000 00000000 00000000 00111010

按位异或（^）

按位异或（^）运算：两个数相同位的二进制数一个为1，一个为0，那么结果的相同位上的数为1，其他的都为0。

x = 00000000 00000000 00000000 00001010

y = 00000000 00000000 00000000 00110010

x ^ y = 00000000 00000000 00000000 00111000

按位取反（~）

按位取反（~）运算：即把当前数中位上为1的设为0，为0的设为1。

a = 00000000 00000000 00000000 00101011

~a = 11111111 11111111 11111111 11010100

按位左移（<<）

按位左移（<<）运算：即把操作数的位数向左移动，每一次移动相当于操作数乘以2。

a = 00000000 00000000 00000000 00000001

a << 3 = 00000000 00000000 00000000 00001000

按位右移（>>）

按位左移（<<）运算：即把操作数的位数向右移动，每一次移动相当于操作数除以2。

a = 00000000 00000000 00000000 00001000

a >> 3 = 00000000 00000000 00000000 00000001

b = 00000000 00000000 00000000 00000001

b >> 3 = 00000000 00000000 00000000 00000000

进制转换

在计算机底层采用的进制数为二进制，而我们平时看到的数字、字符等一般都是十进制，那么为了方便，一般需要进行进制转换。

十进制转二进制

对于整数部分，每次都除以2，余数就为该位的二进制数，这个过程一直进行下去，直到被除数为0为止。然后从左后一个余数读起，直到最前面的余数。如：

对于小数部分，采用乘2取整法，即小数部分乘以2，然后去整数部分，剩下的小数继续乘以2，然后再取整，一直取到小数部分为0为止。如：

0.125转换成二进制

第一步：0.125 * 2 = 0.25 取整数部分为 0，小数部分为 0.25

第二步：0.25 * 2 = 0.5 取整数部分为 0，小数部分为0.5

第三步：0.5 * 2 = 1.0 取整数部分为1，小数部分为0.0

这个时候小数部分为0终止，从第一位开始读数，得：0.001。

注：当小数永远不可能为0时，酌情保留需要位数。

二进制转十进制

二进制转十进制时，进制数的第0位权值是2的0次方，第1位的权值是2的1次方，第2位的权值是2的2次方，依次类推。然后结果相加即得十进制数。其中位数从右向左开始数。

如：110101转十进制

1 * 2^0 = 1

0 * 2^1 = 0

1 * 2^2 = 4

0 * 2^3 = 0

1 * 2^4 = 16

1 * 2^5 = 32

结果相加得：53

原码、补码和反码

原码、补码和反码也是针对于二进制而言的。

原码

即二进制数，其中最高位（从左向右数第一位）是符号位，”0“表示正，”1“表示负，其余位表示数值。

00000000 00000000 00000000 00000100 ：正数4

10000000 00000000 00000000 00000100 ：负数4

反码

正数的反码仍然其本身，如：

00000000 00000000 00000000 00000100 ==> 00000000 00000000 00000000 00000100

负数的反码是对其原码进行逐位取反，0设为1，1设为0，符号位除外。如：

10000000 00000000 00000000 00000100 ==> 11111111 11111111 11111111 11111011

补码

正数的补码也是其本身，如：

00000000 00000000 00000000 00000100 ==> 00000000 00000000 00000000 00000100

负数的补码是在其反码的末位加1，如：

10000000 00000000 00000000 00000100

=反码为=> 11111111 11111111 11111111 11111011

=补码为=> 11111111 11111111 11111111 11111100

小结：正数的原码、反码和补码一样。负数的原码、反码和补码不同，但是计算时之间都有关联性。
